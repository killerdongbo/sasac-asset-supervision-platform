# 企业资产管理系统 — 系统概设文档

> 文档版本：v1.0  
> 更新时间：2026-04-27  
> 作者：知微

---

## 1. 项目背景

企业资产管理系统用于管理企业固定资产、无形资产和软件资产，解决资产登记不统一、流转过程不可追踪、巡检盘点效率低、资产状态不准确、财务折旧和资产处置缺少留痕等问题。

系统基于既有 Axure 原型设计，原型覆盖工作台、资产管理、资产流转、采购、盘点、维保维修、财务折旧、统计预警、系统管理等模块。结合钜哥确认的方向，本项目采用：

- 后端：FastAPI
- 前端：Vue3 + Ant Design Vue
- 数据库：MySQL 8
- 缓存：Redis
- 文件存储：MinIO
- 部署：服务器 124.222.72.18，优先 Docker Compose，后续可扩展 K8s

---

## 2. 系统定位

系统定位为企业内部资产全生命周期管理平台，核心目标是形成资产从登记、入库、使用、巡检、盘点、维修、折旧、处置到清理的完整闭环。

### 2.1 管理对象

| 类型 | 示例 | 管理重点 |
|---|---|---|
| 固定资产 | 电脑、服务器、车辆、办公设备、家具 | 实物位置、使用部门、责任人、状态、折旧 |
| 无形资产 | 专利、著作权、商标 | 权属、有效期、证书、维护状态 |
| 软件资产 | 软件授权、SaaS 订阅、许可证 | 授权数量、到期时间、使用部门、合同附件 |

### 2.2 核心业务主线

本系统的核心业务主线是：

```text
资产基础数据 → 巡检任务 → 巡检结果 → 异常处理
资产基础数据 → 盘点任务 → 盘点结果 → 差异报告 → 差异审批 → 账实调整
```

采购、折旧、维保、报表等模块作为支撑能力逐步扩展。

---

## 3. 总体目标

1. 建立统一资产台账。
2. 形成资产卡片统一入口。
3. 支持资产二维码/条码识别。
4. 支持巡检任务创建、执行、异常上报。
5. 支持盘点任务创建、执行、差异分析和调整。
6. 支持 RBAC 权限管理。
7. 支持审批、日志、附件、报表等企业级能力。
8. 支持客户演示和后续二次扩展。

## 4. 总体架构

```text
浏览器 / 移动端扫码入口
        │
        ▼
Nginx / HTTPS / 静态资源代理
        │
        ├── 前端 Vue3 + Ant Design Vue
        │
        └── 后端 FastAPI /api/v1
                 │
                 ├── MySQL 8：业务数据
                 ├── Redis：缓存、会话、异步任务状态
                 └── MinIO：附件、照片、报表文件
```

---

## 5. 后端架构

后端采用分层结构：

```text
app/
├── api/        # 路由层，负责接口输入输出
├── core/       # 配置、数据库、认证、安全、异常
├── models/     # SQLAlchemy ORM 模型
├── schemas/    # Pydantic 请求/响应模型
├── services/   # 业务逻辑层
├── tasks/      # 定时任务/异步任务
└── main.py     # 应用入口
```

### 5.1 分层职责

| 层 | 职责 |
|---|---|
| API 层 | 参数接收、鉴权依赖、响应封装 |
| Service 层 | 业务规则、状态流转、事务控制 |
| Model 层 | 数据库表映射 |
| Schema 层 | 请求体和响应体校验 |
| Core 层 | 通用基础能力 |

---

## 6. 前端架构

前端采用 Vue3 + Vite + Ant Design Vue。

```text
src/
├── api/          # 请求封装
├── router/       # 路由
├── store/        # Pinia 状态
├── layouts/      # 基础布局
├── views/        # 页面
├── components/   # 通用组件
├── utils/        # 工具函数
└── styles/       # 全局样式
```

### 6.1 UI 规范

- 所有表格使用 Ant Design Vue Table。
- 所有表单使用 Ant Design Vue Form。
- 列表页统一采用：筛选区 + 操作区 + 表格区 + 分页区。
- 详情页统一采用：基础信息卡片 + Tabs。
- 弹窗统一使用 Modal/Drawer。
- 状态字段统一使用 Tag。

## 8. 非功能需求

| 项目 | 要求 |
|---|---|
| 性能 | 常规接口 500ms 内响应，复杂报表 3s 内响应 |
| 安全 | JWT、密码哈希、接口权限、操作日志 |

## 9. 里程碑

| 阶段 | 目标 | 验收标准 |
|---|---|---|
| M1 | 基础底座 | 可登录，能维护用户、角色、资产基础数据 |
| M2 | 巡检盘点 | 可创建任务、执行、生成结果和差异 |
| M3 | 流转审批 | 可处理资产流转和审批 |
| M4 | 财务报表 | 可折旧、统计、导出 |

---

## 10. 风险控制

1. 原型页面多：按阶段拆解，避免一次性铺开。
2. 业务状态复杂：统一状态枚举和状态机。
3. 巡检/盘点数据量大：列表分页、索引优化、异步导出。
4. 财务数据敏感：折旧和调整必须审批留痕。
5. AI 开发中断：出现异常必须提醒钜哥。
