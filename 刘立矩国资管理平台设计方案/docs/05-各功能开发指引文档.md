# 企业资产管理系统 — 各功能开发指引文档

> 文档版本：v1.0  
> 更新时间：2026-04-27  
> 作者：知微

---

## 1. 开发总原则

1. 先完成设计文档，再进入开发。
2. 每次只推进一个明确模块，避免并发写入冲突。
3. 最大并发任务调用数量不得超过 10 个。
4. 若任务异常中断，必须主动提醒钜哥。
5. 完成阶段性版本必须主动汇报。
6. 所有接口必须可在 `/docs` 中查看和调试。
7. 所有核心数据变更必须写入操作日志或生命周期事件。

---

## 2. 后端开发规范

### 2.1 目录结构

```text
backend/app/
├── api/          # 路由层
├── core/         # 配置、数据库、认证、安全、异常
├── models/       # SQLAlchemy 模型
├── schemas/      # Pydantic 模型
├── services/     # 业务逻辑
├── tasks/        # 定时/异步任务
└── main.py
```

### 2.2 新增一个功能的标准步骤

以“资产分类”为例：

1. 在 `models/asset.py` 中定义 ORM 模型。
2. 在 `schemas/asset.py` 中定义请求/响应模型。
3. 在 `services/asset_service.py` 中写业务逻辑。
4. 在 `api/asset.py` 中暴露接口。
5. 增加权限编码，如 `asset_category:create`。
6. 写 Alembic 迁移。
7. 在 Swagger 中验证接口。

## 4. Phase 1 功能开发指引

### 4.1 登录认证

**目标：** 支持用户登录、token 存储、获取用户信息。

后端：
- `POST /api/v1/auth/login`
- `GET /api/v1/auth/me`

前端：
- `views/auth/Login.vue`
- `store/user.ts`
- `router/guard.ts`

验收：
- 正确账号可登录。
- 错误密码提示。
- 未登录访问业务页自动跳转登录页。
- 登录后左侧菜单按权限展示。

### 4.2 用户管理

**目标：** 支持用户列表、新增、编辑、禁用。

后端：
- `GET /users/list`
- `POST /users`
- `PUT /users/{id}`
- `DELETE /users/{id}`

前端：
- `views/system/user/UserList.vue`
- `api/user.ts`

验收：
- 支持按用户名/姓名搜索。
- 支持分页。
- 支持新增和编辑。
- 禁用用户不能登录。

### 4.3 角色权限管理

**目标：** 支持角色维护和权限分配。

后端：
- `GET /users/roles`
- `POST /users/roles`
- `PUT /users/roles/{id}`
- `GET /users/permissions`

前端：
- `views/system/role/RoleList.vue`
- 权限树组件 `PermissionTree.vue`

验收：
- 可创建角色。
- 可给角色分配菜单/按钮/API 权限。
- 修改角色权限后重新登录生效。

### 4.4 资产分类管理

**目标：** 支持资产分类树维护。

后端：
- `GET /assets/categories`
- `POST /assets/categories`
- `PUT /assets/categories/{id}`
- `DELETE /assets/categories/{id}`

前端：
- `views/asset/category/CategoryList.vue`

验收：
- 分类支持树形展示。
- 支持固定资产、无形资产、软件资产分类。
- 已被资产使用的分类不允许删除。

### 4.5 存放位置管理

**目标：** 支持地点/楼栋/楼层/房间树维护。

后端：
- `GET /assets/locations`
- `POST /assets/locations`
- `PUT /assets/locations/{id}`
- `DELETE /assets/locations/{id}`

前端：
- `views/asset/location/LocationList.vue`

验收：
- 支持树形位置。
- 支持资产登记时选择位置。

### 4.6 资产登记与清单

**目标：** 支持资产新增、编辑、查询、查看详情。

后端：
- `GET /assets`
- `POST /assets`
- `GET /assets/{id}`
- `PUT /assets/{id}`
- `DELETE /assets/{id}`

前端：
- `views/asset/list/AssetList.vue`
- `views/asset/form/AssetFormDrawer.vue`
- `views/asset/detail/AssetDetail.vue`

验收：
- 新增资产自动生成编号。
- 支持资产类型切换。
- 支持扩展属性。
- 资产详情展示基础信息和 Tabs。

## 5. Phase 2 巡检功能开发指引

### 5.1 巡检任务创建

后端建议文件：

```text
models/inspection.py
schemas/inspection.py
services/inspection_service.py
api/inspection.py
```

核心逻辑：

1. 根据范围筛选资产。
2. 创建 `inspection_task`。
3. 批量生成 `inspection_item`。
4. 初始化任务统计字段。

验收：
- 可按分类/位置/部门创建任务。
- 创建后明细数量正确。
- 巡检任务状态为待开始。

### 5.2 巡检执行

核心逻辑：

1. 巡检人员进入“我的巡检”。
2. 扫码或点击资产。
3. 提交结果：正常/异常/跳过。
4. 更新明细状态和任务统计。
5. 异常项自动写入 `inspection_abnormal`。

验收：
- 支持上传照片。
- 支持异常说明。
- 全部明细完成后任务可完成。

### 5.3 巡检报告

内容：

- 任务基本信息
- 总资产数
- 已巡检数
- 异常数
- 异常清单
- 巡检人和完成时间

验收：
- 报告数据与明细一致。
- 支持导出 Excel/PDF（后续）。

## 6. Phase 2 盘点功能开发指引

### 6.1 盘点任务创建

建议文件：

```text
models/inventory.py
schemas/inventory.py
services/inventory_service.py
api/inventory.py
```

核心逻辑：

1. 选择盘点范围。
2. 创建 `inventory_task`。
3. 生成账面盘点明细。
4. 状态进入草稿或待审批。

验收：
- 支持按部门/位置/分类创建盘点任务。
- 明细数量与资产范围一致。

### 6.2 盘点执行

核心逻辑：

1. 扫描资产二维码。
2. 根据资产编号匹配账面明细。
3. 记录实际位置、状态、照片。
4. 系统自动判断：正常、盘盈、盘亏、位置不符、状态不符。

验收：
- 已盘资产不重复统计。
- 未扫描到账面资产标记为盘亏。
- 扫描到账面外资产标记为盘盈。

### 6.3 差异报告与调整

核心逻辑：

1. 生成差异报告。
2. 盘点负责人填写差异原因。
3. 提交审批。
4. 审批通过后执行调整。
5. 写入生命周期事件和操作日志。

验收：
- 差异类型统计准确。
- 调整动作可追溯。
- 未审批不得直接变更资产账面数据。

## 7. Phase 3 功能开发指引

### 7.1 资产流转

围绕资产全生命周期主线建设：采购 → 入库 → 领用 → 调拨 → 盘点 → 报废。

开发重点：
- 入库、领用、调拨、报废均需有独立菜单入口，也可复用统一流转单据表。
- 表单不得要求用户手填内部 ID，应通过资产、位置、部门、用户下拉选择。
- 入库完成后更新资产位置/部门和状态；领用完成后更新责任人/部门/位置和状态；调拨完成后更新位置/部门/责任人；报废完成后资产退出可用状态。
- 已报废资产不得再领用、调拨、入库。
- 审批可作为正式流程，演示阶段允许管理员直接执行待审批单据，但必须保留状态和日志。
- 写入生命周期事件。

验收：
- 四个菜单均可新增、查看和执行单据。
- 单据列表展示资产编号/名称、目标位置/部门/人员中文名称，不展示裸 ID。
- 执行后资产清单和资产详情生命周期同步变化。
- 操作失败时有明确业务提示。

### 7.2 采购管理

开发重点：
- 采购申请支持多明细。
- 采购验收后可生成资产。
- 合同、发票、验收单作为附件。

### 7.3 维保维修

开发重点：
- 巡检异常可转维修单。
- 维保可由到期预警触发。
- 维修完成后更新资产状态。

## 8. Phase 4 功能开发指引

### 8.1 折旧

开发重点：
- 折旧规则按资产分类配置。
- 每月定时计提。
- 折旧记录确认后不可直接删除。

### 8.2 报表

开发重点：
- 资产分类统计。
- 部门资产统计。
- 巡检异常统计。
- 盘点差异统计。
- 折旧汇总。

### 8.3 预警

开发重点：
- 维保到期预警。
- 巡检超期预警。
- 借用超期预警。
- 闲置资产预警。

## 9. 测试与验收标准

### 9.1 后端

- FastAPI 启动无报错。
- `/health` 返回正常。
- `/docs` 可访问。
- 核心接口请求/响应符合 API 文档。
- Alembic 可生成迁移。

### 9.2 前端

- 登录页可用。
- 主布局可用。
- 菜单按权限展示。
- 资产列表、资产表单、资产详情可用。
- 表单校验完整。

### 9.3 业务

- 可创建资产。
- 可创建巡检任务并提交结果。
- 可创建盘点任务并生成差异报告。
- 差异调整必须留痕。

---

## 10. 开发顺序建议

1. 整理现有后端代码，使其与文档一致。
2. 完成数据库迁移。
3. 初始化前端项目。
4. 完成登录和基础布局。
5. 完成用户/角色/权限。
6. 完成资产分类、位置、资产登记。
7. 完成巡检任务。
8. 完成盘点任务。
9. 再进入采购、流转、维保、折旧。
