## 3. 认证与权限模块

### 3.1 模块职责

- 用户登录
- 获取当前用户信息
- 用户管理
- 角色管理
- 权限管理
- 部门管理
- 前端菜单权限控制
- 后端接口权限控制

### 3.2 RBAC 模型

```text
User 用户
  └── UserRole 用户角色关联
        └── Role 角色
              └── RolePermission 角色权限关联
                    └── Permission 权限
```

### 3.3 登录流程

```text
输入用户名密码
  → 后端校验密码 bcrypt
  → 校验用户状态
  → 签发 JWT
  → 返回 token 和用户基础信息
```

### 3.4 权限控制规则

1. 菜单权限：控制前端路由是否展示。
2. 按钮权限：控制新增、编辑、删除、导出等按钮。
3. 接口权限：控制后端接口是否允许访问。
4. 超级管理员拥有全部权限。

## 5. 巡检模块

### 5.1 模块职责

- 创建巡检任务
- 指定巡检范围
- 指定巡检人员
- 生成巡检明细
- 执行巡检
- 上报异常
- 生成巡检报告

### 5.2 巡检任务状态

| 状态 | 编码 | 说明 |
|---|---|---|
| 草稿 | draft | 尚未发布 |
| 待开始 | pending | 已发布未开始 |
| 进行中 | processing | 巡检执行中 |
| 已完成 | completed | 全部巡检完成 |
| 已关闭 | closed | 管理员关闭 |

### 5.3 巡检明细状态

| 状态 | 编码 | 说明 |
|---|---|---|
| 待巡检 | pending | 尚未检查 |
| 正常 | normal | 巡检正常 |
| 异常 | abnormal | 发现异常 |
| 跳过 | skipped | 特殊原因跳过 |

### 5.4 巡检流程

```text
资产管理员创建任务
  → 选择资产范围/分类/位置/部门
  → 分配巡检人员
  → 系统生成巡检明细
  → 巡检人员扫码或列表执行
  → 填写结果、照片、备注
  → 异常项进入异常处理
  → 任务完成后生成报告
```

### 5.5 巡检异常处理

异常类型包括：

- 资产不存在
- 位置不一致
- 责任人不一致
- 资产损坏
- 无法使用
- 缺少附件/证书
- 其他异常

异常可以转为维修单或处理记录。

## 6. 盘点模块

### 6.1 模块职责

- 创建盘点任务
- 指定盘点范围
- 分配盘点人员
- 生成盘点清单
- 执行盘点
- 账实比对
- 生成差异报告
- 差异审批与调整

### 6.2 盘点任务状态

| 状态 | 编码 | 说明 |
|---|---|---|
| 草稿 | draft | 尚未发布 |
| 待审批 | pending_approval | 等待审批 |
| 进行中 | processing | 正在盘点 |
| 待确认 | pending_confirm | 等待差异确认 |
| 已完成 | completed | 盘点完成 |
| 已关闭 | closed | 关闭 |

### 6.3 盘点结果类型

| 类型 | 编码 | 说明 |
|---|---|---|
| 正常 | normal | 账实一致 |
| 盘盈 | surplus | 实物多于账面 |
| 盘亏 | deficit | 账面有但实物缺失 |
| 位置不符 | location_mismatch | 实际位置与账面不符 |
| 状态不符 | status_mismatch | 实际状态与账面不符 |

### 6.4 盘点流程

```text
创建盘点任务
  → 选择范围：分类/部门/位置/资产
  → 审批后发布
  → 生成盘点明细
  → 盘点人员扫码/批量录入
  → 系统自动比对账实
  → 生成差异报告
  → 差异原因填写
  → 审批通过后调整资产状态/位置/责任人
```

### 6.5 差异调整规则

1. 盘亏不得直接删除资产，只能将资产标记为待处理或待处置。
2. 盘盈需补充资产登记信息后入账。
3. 位置不符可在审批通过后更新资产位置。
4. 状态不符可在审批通过后更新资产状态。
5. 所有调整必须写入资产生命周期事件和操作日志。

## 7. 资产流转模块

### 7.1 覆盖流程

- 采购验收
- 资产入库
- 资产领用
- 资产调拨
- 资产盘点
- 资产报废

资产全生命周期主线：

```text
采购申请 → 验收入库 → 资产入库 → 资产领用 → 资产调拨 → 资产盘点 → 资产报废
```

说明：
- 采购验收可直接生成资产并写入“采购验收入库”生命周期事件。
- 入库管理用于把新购、退回或未分配资产纳入库房，更新资产位置、部门和状态。
- 资产领用用于把库房资产分配给部门或责任人，更新责任人、部门、位置和状态。
- 资产调拨用于跨位置、跨部门或跨责任人转移资产，必须记录调拨前后信息。
- 盘点通过盘点任务核验账实一致性，差异调整必须审批后执行。
- 报废用于资产达到处置条件后退出可用台账，状态更新为已报废，后续不得再领用/调拨。

### 7.2 通用流转状态

```text
草稿 → 待审批 → 审批通过 → 执行中 → 已完成
              ↘ 审批驳回
```

### 7.3 关键规则

- 资产处于维修、报废、处置状态时不得借用/调拨。
- 资产调拨必须同时更新部门、位置和责任人。
- 流转完成后生成生命周期事件。
- 入库、领用、调拨、报废均应形成独立业务单据，可走审批，也允许管理员在演示/简化模式下执行单据。
- 入库完成后资产状态为正常或闲置；领用完成后资产状态为借用中/使用中；调拨不改变资产可用状态；报废完成后资产状态为已报废。
- 已报废资产不得再次入库、领用、调拨或参与新巡检/盘点范围。

## 8. 采购模块

### 8.1 流程

```text
采购申请 → 审批 → 采购订单 → 到货验收 → 入库 → 资产登记/关联
```

### 8.2 关键规则

- 采购申请可包含多条明细。
- 采购验收通过后可一键生成资产登记记录。
- 采购附件包括合同、发票、验收单。

---

## 9. 维保维修模块

### 9.1 维保

维保偏计划性，通常由维保周期触发。

```text
维保计划 → 到期提醒 → 维保申请 → 派单 → 完成 → 验收
```

### 9.2 维修

维修偏异常处理，通常由巡检异常、用户报修触发。

```text
报修申请 → 审批/派单 → 维修中 → 完成 → 确认
```

## 10. 财务折旧模块

### 10.1 折旧规则

支持：

- 直线法
- 双倍余额递减法
- 年数总和法

### 10.2 折旧流程

```text
设置折旧规则 → 每月批量计提 → 生成折旧记录 → 财务确认 → 锁定记录
```
