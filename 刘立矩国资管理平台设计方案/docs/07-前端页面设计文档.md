# 企业资产管理系统 — 前端页面设计文档

> 文档版本：v1.0  
> 更新时间：2026-04-27  
> 前端技术栈：Vue3 + Vite + TypeScript + Ant Design Vue + Pinia  
> 作者：知微

---

## 1. 前端设计目标

1. 与原型页面结构保持一致。
2. 统一使用 Ant Design Vue，保证客户试用/演示界面观感统一。
3. 优先完成 Phase 1 和 Phase 2 页面。
4. 所有列表页、表单页、详情页保持统一交互模式。
5. 支持权限控制、动态菜单和按钮级权限。

## 4. 通用页面模板

### 4.1 列表页模板

所有列表页统一采用：

```text
PageContainer
  ├─ SearchForm 筛选区
  ├─ Toolbar 操作按钮区
  ├─ ATable 数据表格
  └─ Pagination 分页
```

通用交互：

- 点击“查询”刷新第一页。
- 点击“重置”清空筛选条件。
- 新增/编辑优先使用 Drawer。
- 删除必须二次确认。
- 状态字段使用 Tag。
- 长文本使用 Tooltip。

### 4.2 表单抽屉模板

```text
ADrawer
  ├─ AForm
  ├─ 表单项
  └─ Footer：取消/保存
```

规则：

- 新增和编辑共用一个组件。
- 保存成功后关闭抽屉并刷新列表。
- 表单校验失败时保留当前输入。

### 4.3 详情页模板

```text
PageContainer
  ├─ Header 操作区
  ├─ ADescriptions 基础信息
  └─ ATabs
       ├─ 基础信息
       ├─ 生命周期
       ├─ 巡检记录
       ├─ 盘点记录
       ├─ 维保维修
       ├─ 附件
       └─ 操作日志
```

## 6. 页面详细设计

### 6.1 登录页

组件：

- `a-card`
- `a-form`
- `a-input`
- `a-input-password`
- `a-button`

字段：

| 字段 | 规则 |
|---|---|
| username | 必填 |
| password | 必填，最少 6 位 |

交互：

1. 点击登录调用 `/auth/login`。
2. 保存 token。
3. 调用 `/auth/me` 获取用户权限。
4. 跳转 `/dashboard`。

### 6.2 用户管理页

筛选项：用户名/姓名、部门、状态。

表格列：

| 列 | 说明 |
|---|---|
| username | 用户名 |
| real_name | 姓名 |
| department | 部门 |
| phone | 手机号 |
| is_active | 状态 |
| created_at | 创建时间 |
| actions | 编辑/禁用/删除 |

按钮：新增、编辑、删除、重置密码。

### 6.3 资产清单页

筛选项：关键词、资产类型、分类、状态、部门、位置。

表格列：

| 列 | 说明 |
|---|---|
| asset_code | 资产编号 |
| asset_name | 资产名称 |
| asset_type | 资产类型 |
| category | 分类 |
| status | 状态 |
| location | 存放位置 |
| department | 使用部门 |
| responsible_user | 责任人 |
| purchase_price | 购入价格 |
| actions | 查看/编辑/删除 |

操作：

- 新增资产
- 导入
- 导出
- 打印二维码

### 6.4 资产表单

字段分组：

1. 基础信息：名称、类型、分类、规格、厂家。
2. 购置信息：购入日期、购入价格、供应商。
3. 使用信息：位置、部门、责任人、状态。
4. 扩展信息：无形资产/软件资产专属字段。
5. 附件信息：发票、合同、照片、证书。

不同资产类型的扩展字段：

| 类型 | 扩展字段 |
|---|---|
| 固定资产 | 规格型号、序列号、保修期 |
| 无形资产 | 证书号、权利人、有效期 |
| 软件资产 | 授权数量、授权方式、到期日期 |

### 6.5 资产详情页

Tabs：

1. 基础信息
2. 生命周期
3. 巡检记录
4. 盘点记录
5. 维保维修
6. 财务信息
7. 附件
8. 操作日志

### 6.6 巡检任务列表页

筛选项：任务名称、状态、负责人、计划日期。

表格列：任务编号、任务名称、状态、总数、已完成、异常数、负责人、计划时间、操作。

操作：新建、发布、开始、查看、关闭。

### 6.7 巡检执行页

布局：

```text
任务信息卡片
进度条
资产巡检明细表
巡检结果抽屉
```

巡检结果字段：

- 结果：正常/异常/跳过
- 实际位置
- 实际状态
- 异常类型
- 异常描述
- 照片

### 6.8 资产全生命周期流转页

菜单包含：入库管理、资产领用、资产调拨、资产报废、流转单据。

列表列：

- 序号
- 单据编号
- 业务类型
- 资产编号/资产名称
- 目标位置
- 目标部门
- 目标使用人
- 状态
- 创建时间
- 操作

新增抽屉：

- 资产：下拉选择，展示资产编号、名称、当前位置、当前状态。
- 目标位置：下拉选择。
- 目标部门：下拉选择。
- 目标使用人：领用/调拨时可选。
- 原因/说明：文本域。

交互规则：

- 页面不得出现需要用户手填的资产 ID、位置 ID、部门 ID、用户 ID。
- 入库、领用、调拨、报废菜单自动固定对应业务类型。
- 报废菜单只允许选择未报废资产。
- 执行完成后刷新列表，并可在资产详情生命周期中看到事件。

### 6.8 盘点任务列表页

筛选项：任务名称、状态、负责人、盘点日期。

表格列：任务编号、任务名称、状态、总数、已盘、正常、盘盈、盘亏、差异、操作。

### 6.9 盘点执行页

功能：

- 扫码输入资产编号
- 手动录入资产编号
- 显示账面信息
- 填写实际位置/状态
- 系统自动判断差异

### 6.10 盘点差异报告页

统计卡片：

- 总资产数
- 已盘点数
- 正常数
- 盘盈数
- 盘亏数
- 位置不符数
- 状态不符数

表格列：资产编号、资产名称、差异类型、账面位置、实际位置、账面状态、实际状态、差异说明、调整操作。

## 99. 交互设计强制规则补充

- 所有功能菜单对应的新增、编辑、详情页、详情查询类页面，统一使用右侧抽屉弹出样式承载。
- 前端实现统一采用 Ant Design Vue `ADrawer`，默认从右侧滑出；基础维护表单建议宽度 `720px` 到 `760px`，资产登记、审批、详情等复杂表单建议宽度 `860px` 或按内容扩展。
- 抽屉内表单默认采用两列网格布局，长文本、权限选择、备注、详情说明等宽字段应跨两列展示；移动端自动退化为单列。
- 抽屉底部统一放置操作区，基础按钮为“取消”和“保存”；详情只读抽屉可仅保留“关闭”。
- 业务表单、详情查看、查询详情不得使用居中 `AModal`；`AModal` 仅保留用于删除、禁用、撤销等轻量二次确认。
- 已有功能页面需按本规则改造，后续新增页面默认遵守本规则。
