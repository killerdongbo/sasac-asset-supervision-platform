## 1. 设计说明

数据库设计围绕“资产台账 + 巡检 + 盘点”展开，兼顾采购、流转、维保、维修、折旧、审批、日志等后续扩展能力。

### 1.1 命名规范

- 表名：小写下划线，如 `asset_item`
- 主键：统一 `id BIGINT AUTO_INCREMENT`
- 外键字段：`xxx_id`
- 时间字段：`created_at`、`updated_at`
- 金额字段：`DECIMAL(18,2)`
- 状态字段：`VARCHAR(32)`，业务枚举由应用层控制
- 是否字段：`TINYINT(1)`

### 1.2 通用字段

建议核心业务表统一包含：

```sql
created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
created_by BIGINT NULL COMMENT '创建人ID',
updated_by BIGINT NULL COMMENT '更新人ID',
remarks VARCHAR(500) NULL COMMENT '备注'
```

## 2. 系统权限表

### 2.1 用户表 sys_user

```sql
CREATE TABLE sys_user (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '用户ID',
  username VARCHAR(64) NOT NULL COMMENT '用户名',
  password_hash VARCHAR(255) NOT NULL COMMENT '密码哈希',
  real_name VARCHAR(64) NULL COMMENT '真实姓名',
  email VARCHAR(128) NULL COMMENT '邮箱',
  phone VARCHAR(32) NULL COMMENT '手机号',
  department_id BIGINT NULL COMMENT '部门ID',
  is_active TINYINT(1) NOT NULL DEFAULT 1 COMMENT '是否启用',
  last_login_at DATETIME NULL COMMENT '最后登录时间',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  created_by BIGINT NULL,
  updated_by BIGINT NULL,
  UNIQUE KEY uk_sys_user_username (username),
  KEY idx_sys_user_department (department_id)
) COMMENT='系统用户表';
```

## 4. 巡检表

### 4.1 巡检任务 inspection_task

```sql
CREATE TABLE inspection_task (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  task_code VARCHAR(64) NOT NULL COMMENT '巡检任务编号',
  task_name VARCHAR(255) NOT NULL COMMENT '任务名称',
  task_status VARCHAR(32) NOT NULL DEFAULT 'draft' COMMENT '任务状态',
  scope_type VARCHAR(32) NOT NULL COMMENT '范围类型 all/category/location/department/custom',
  scope_config JSON NULL COMMENT '范围配置',
  start_date DATE NULL COMMENT '计划开始日期',
  end_date DATE NULL COMMENT '计划结束日期',
  assignee_id BIGINT NULL COMMENT '负责人ID',
  total_count INT NOT NULL DEFAULT 0 COMMENT '总数量',
  completed_count INT NOT NULL DEFAULT 0 COMMENT '已完成数量',
  abnormal_count INT NOT NULL DEFAULT 0 COMMENT '异常数量',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  created_by BIGINT NULL,
  remarks VARCHAR(500) NULL,
  UNIQUE KEY uk_inspection_task_code (task_code),
  KEY idx_inspection_status (task_status),
  KEY idx_inspection_assignee (assignee_id)
) COMMENT='巡检任务表';
```

### 4.2 巡检明细 inspection_item

```sql
CREATE TABLE inspection_item (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,

  actual_status VARCHAR(32) NULL COMMENT '实际状态',
  abnormal_type VARCHAR(64) NULL COMMENT '异常类型',
  abnormal_desc VARCHAR(1000) NULL COMMENT '异常描述',
  photos JSON NULL COMMENT '照片',
  inspected_by BIGINT NULL COMMENT '巡检人',
  inspected_at DATETIME NULL COMMENT '巡检时间',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_inspection_task_asset (task_id, asset_id),

  KEY idx_inspection_item_asset (asset_id),
  KEY idx_inspection_item_status (item_status)

```

## 9. 索引设计原则

1. 所有编号字段建立唯一索引。
2. 所有状态字段建立普通索引。
3. 所有外键关联字段建立普通索引。
4. 巡检/盘点明细按任务 ID 和资产 ID 建组合索引。
5. 日志表按时间和业务对象建索引。
