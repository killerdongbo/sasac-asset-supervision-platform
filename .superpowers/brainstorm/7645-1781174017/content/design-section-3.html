设计第三节：技术选型 & 部署架构

SaaS多租户 · 信创兼容 · 7月MVP交付

🧱 技术栈建议

A

Java/SpringBoot 全栈

后端 SpringBoot 3 + MyBatis-Plus + PostgreSQL；前端 Vue3 + Element Plus；信创生态最成熟，政务项目标准选择

B

Go 轻量架构

后端 Go + Gin + GORM + PostgreSQL；前端 Vue3；性能优秀、部署简单、资源占用低

C

混合技术栈

核心业务 Java（信创合规） + 数据服务 Go（高性能） + 前端 Vue3；各自发挥优势，但运维复杂度增加

📋 推荐方案A详细技术选型

层次技术选型选型理由
前端框架Vue 3 + TypeScript + Element Plus政务项目主流选型，组件库成熟，团队招聘容易
可视化/驾驶舱ECharts 5 + DataV国产可视化库，大屏适配成熟，信创兼容
后端框架Spring Boot 3.x + MyBatis-Plus政务信创标配，生态最完善
数据库PostgreSQL 15 + PostGIS（空间数据扩展）开源、信创兼容，PostGIS支持土地/房产空间数据
缓存Redis 7Session共享、租户配置缓存、驾驶舱数据预热
文件存储MinIO（对象存储）资产照片、附件、报表导出文件，兼容S3协议
报表引擎JasperReports + Apache POI复杂报表模板（19类）+ Excel导入导出
消息队列RabbitMQ（MVP可选，后续引入）异步任务：折旧批量计提、报表生成、数据同步
API网关Spring Cloud Gateway统一鉴权、租户注入、限流、日志
容器化Docker + Docker Compose（MVP）MVP阶段够用，后续可升级K8s
信创适配达梦/人大金仓（PG兼容）+ 麒麟OS + 东方通基于PG协议兼容，迁移成本低
