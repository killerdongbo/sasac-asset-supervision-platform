# 湛江国资监管平台二期 — 四新模块实施设计

> 日期：2026-06-17 | 版本：V1.0 | 状态：已确认

## 一、总体框架

二期建设分两大阶段、12 个里程碑，总工期 22 周，投入 **后端 1 人 + 前端 1 人**。

```
Phase 1（W1-W10）：四新模块（HR / 三重一大 / 项目 / 投资）

  W1-2   M1 — HR基础花名册（组织 / 人员 / 异动 / 合同）
  W3-4   M2 — 三重一大（决策 / 会议 / 督办 / 留痕）
  W5-6   M3 — 项目管理（立项 / 计划 / 预算 / 进度 / 验收）
  W7-8   M4 — 投资管理（投前 / 投中 / 投后 / 股权穿透 / 退出）
  W9-10  M5 — HR深度人事（薪酬 / 考勤 / 绩效 / 招聘 / 培训 / 成本）

Phase 2（W11-W22）：六升级 + 移动端重做

  W11-12 M6  — 产权管理
  W13-14 M7  — 财务监督
  W15-16 M8  — 业绩考核与薪酬
  W17-18 M9  — 监督追责
  W19-20 M10 — 重大事项管理
  W21-22 M11 — 预警驾驶舱 + 移动端重做
```

每模块 2 周内交付完整可验收的功能：数据库设计 → API 开发 → 前端页面 → 联调测试。

## 二、模块依赖与数据流

```
一期底座（已有）
  ├─ 组织架构树 ─────────────────→ HR基础（继承）
  │   └─ 部门 / 岗位 / 人员账号 ──→ 所有新模块（统一身份）
  ├─ RBAC 权限体系 ──────────────→ 所有新模块（继承）
  ├─ 工作流审批引擎 ─────────────→ 三重一大 + 项目 + 投资（复用）
  └─ 审计日志 / 操作追溯 ────────→ 所有新模块（继承）

Phase 1 四新模块之间：
  HR基础(W1-2) ──组织+人员数据──→ 三重一大(W3-4)
                                   ├─ 决策结果 ──→ 项目管理(W5-6)
                                   │              └─ 项目立项需三重一大审批记录
                                   ├─ 决策结果 ──→ 投资管理(W7-8)
                                   │              └─ 投资决策需三重一大上会
                                   └─ 大额资金决策记录 ──→ 投资管理

  HR深度(W9-10) ←──薪酬数据来自── HR基础的人员档案
```

> 关键设计决策：HR 基础花名册做完后，三重一大 / 项目 / 投资都可以独立开发，不互相阻塞。串行的原因是资源约束（1+1），不是技术依赖。

## 三、数据库设计策略

**采用方案 A：扩展现有 Schema。**

一期已是单 Schema 多表模式，代码按包（`asset/`、`supervision/`）隔离。二期模块大量关联一期数据（组织、人员、审批），同库 JOIN 避免性能问题。通过独立包结构保证模块边界清晰。

新表命名规范：

| 模块 | 表前缀 | 示例 |
|------|--------|------|
| HR基础 | `hr_` | `hr_employee`、`hr_employee_change`、`hr_contract` |
| HR深度 | `hr_` | `hr_salary`、`hr_attendance`、`hr_performance` |
| 三重一大 | `decision_` | `decision_item`、`decision_meeting`、`decision_resolution`、`decision_supervision` |
| 项目管理 | `pm_` | `pm_project`、`pm_budget`、`pm_progress`、`pm_contract` |
| 投资管理 | `invest_` | `invest_plan`、`invest_project`、`invest_deal`、`invest_equity_tree` |

每个模块新增 4~7 张业务表，不影响一期表结构。

## 四、代码目录结构

```
server/src/main/java/com/sasac/platform/
  hr/          → controller / service / entity / mapper / dto
  decision/    → controller / service / entity / mapper / dto
  project/     → controller / service / entity / mapper / dto
  investment/  → controller / service / entity / mapper / dto

web/src/
  api/hr.ts, decision.ts, project.ts, investment.ts
  views/hr/         → EmployeeList、SalaryManage、PerformanceReview …
  views/decision/   → ItemApply、MeetingManage、ResolutionTrack …
  views/project/    → ProjectList、BudgetControl、GanttView …
  views/investment/ → PortfolioList、EquityTree、InvestMonitor …
  router/index.ts   → 新路由挂载
```

API 格式继承一期统一规范：

```typescript
interface ApiResponse<T> {
  success: boolean
  data?: T
  error?: string
  meta?: { total: number; page: number; limit: number }
}
```

多租户通过 `X-Tenant-Id` Header 注入，全程数据隔离。

## 五、逐模块交付清单

### M1 — HR 基础花名册（W1-2）

**数据库**：`hr_organization`、`hr_employee`、`hr_employee_change`、`hr_contract`

| 层级 | 交付内容 |
|------|---------|
| 后端 API | 组织树 CRUD、人员档案 CRUD、异动流程（入职/转正/调动/离职）、合同管理（签订/续签/到期预警）|
| 前端页面 | 组织架构树（树形控件）、人员档案列表 + 详情卡片、异动申请 + 审批流、合同台账 + 到期预警列表 |
| 对接一期 | 继承一期 `sys_org` 组织树、`sys_user` 账号体系；人员异动触发审批流引擎 |

### M2 — 三重一大（W3-4）

**数据库**：`decision_item`、`decision_meeting`、`decision_resolution`、`decision_supervision`

| 层级 | 交付内容 |
|------|---------|
| 后端 API | 事项申报 + 分类、审核流程、会议管理（创建/通知/纪要）、决议发布、督办任务分派 + 跟踪 |
| 前端页面 | 事项申报表单、审核列表（待审/已审）、会议管理（议题征集→纪要录入→决议生成）、督办看板（进度 + 超期标记）|
| 对接 | 复用审批流引擎；大额资金事项关联财务监督模块（Phase 2）数据校验位；对接 HR 人员数据 |

### M3 — 项目管理（W5-6）

**数据库**：`pm_project`、`pm_budget`、`pm_progress`、`pm_contract`、`pm_document`、`pm_acceptance`

| 层级 | 交付内容 |
|------|---------|
| 后端 API | 立项 CRUD + 审批、预算编制 + 变更控制、进度填报（周报/月报）、甘特图数据接口、合同台账、验收评价 |
| 前端页面 | 项目列表 + 全景看板、甘特图进度视图、预算执行对比图、合同履约跟踪、验收表单 |
| 对接 | 项目立项关联三重一大审批记录；项目验收关联投资管理模块（M4）的投后评价 |

### M4 — 投资管理（W7-8）

**数据库**：`invest_plan`、`invest_project`、`invest_dd`、`invest_deal`、`invest_post`、`invest_exit`、`invest_equity_tree`

| 层级 | 交付内容 |
|------|---------|
| 后端 API | 投资计划 CRUD、项目库管理、投前尽调管理、交易管理、投后监控、股权穿透树、退出管理、负面清单校验 |
| 前端页面 | 投资组合列表 + 全景看板、股权穿透树图（ECharts 树图）、投后监控仪表盘、退出管理流程 |
| 对接 | 投资决策关联三重一大；投后数据关联财务监督；股权树联动产权管理 |

### M5 — HR 深度人事（W9-10）

**数据库**：`hr_salary`、`hr_attendance`、`hr_performance`、`hr_recruitment`、`hr_training`

| 层级 | 交付内容 |
|------|---------|
| 后端 API | 薪酬计算引擎、考勤规则配置 + 打卡数据接入、绩效考核方案 + 评分汇总、招聘流程管理、培训课程 + 学分 |
| 前端页面 | 工资条生成与推送、考勤日历 + 请假加班审批、绩效评分表 + 结果看板、招聘管道看板、培训课程列表 |

## 八、约束与假设

- 资源：后端 1 人 + 前端 1 人，串行开发
- 技术栈：继承一期 SpringBoot 3 + Vue 3 + PostgreSQL 15 + Element Plus
- 开发模式：TDD（测试驱动开发），单元测试覆盖率 ≥ 80%
- 每模块完成后进行代码审查
- Phase 2 细粒度设计在 Phase 1 推进中迭代确认
