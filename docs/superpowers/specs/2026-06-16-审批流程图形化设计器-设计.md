> 版本：v1.0  
> 日期：2026-06-16  
> 状态：已确认，待实施

## 1. 概述

将现有线性表单式审批流程配置重构为基于 Vue Flow 的图形化流程设计器，支持拖拽式创建含分支、判断、并行等复杂逻辑的审批流程。

### 1.1 核心目标

- 用 Vue Flow 图形编辑器替代现有表单式节点配置
- 7 种节点类型：开始、审批、判断、并行、汇聚、子流程、结束
- 3 张新表替代现有 5 张审批表
- 后端执行引擎支持条件分支、并行流转、子流程递归
- 保留旧 API 共存，验证后切换

### 3.4 graph_json 结构

```json
{
  "nodes": [
    { "id": "start1",  "type": "start",    "label": "开始",
      "position": {"x":400, "y":50} },
    { "id": "node1",   "type": "approval", "label": "部门负责人审批",
      "data": { "approverRole": "DEPT_HEAD", "approveMode": "SINGLE",
                "timeoutHours": 24, "timeoutAction": "ESCALATE",
                "allowAddSign": true, "canReject": true }},
    { "id": "gate1",   "type": "gateway",  "label": "金额>10万?",
      "data": { "conditionExpr": "amount > 100000" }},
    { "id": "fork1",   "type": "parallel", "label": "并行审批" },
    { "id": "join1",   "type": "join",     "label": "汇合",
      "data": { "mergeMode": "ALL" }},
    { "id": "sub1",    "type": "subflow",  "label": "采购子流程",
      "data": { "refDefId": 123 }},
    { "id": "end1",    "type": "end",      "label": "结束" }
  ],
  "edges": [
    { "id": "e1", "source": "start1", "target": "node1" },
    { "id": "e2", "source": "node1",  "target": "gate1" },
    { "id": "e3", "source": "gate1",  "target": "end1",  "label": "否" },
    { "id": "e4", "source": "gate1",  "target": "fork1", "label": "是" }
  ]
}
```

## 4. 7 种节点类型

| 节点 | 样式 | 入度 | 出度 | 可配置属性 |
|------|------|------|------|-----------|
| start | 蓝圆角 | 0 | 1 | 标签 |
| approval | 灰方角 | ≥1 | 1 | 审批角色、模式(SINGLE/COUNTER_SIGN/OR_SIGN)、超时小时、超时动作、允许加签、允许驳回、升级角色 |
| gateway | 黄菱形 | ≥1 | 2-N | 条件表达式 |
| parallel | 紫方角 | 1 | 2-N | 标签 |
| join | 紫方角 | 2-N | 1 | mergeMode(ALL/ANY) |
| subflow | 粉方角 | ≥1 | 1 | 引用流程定义 ID |
| end | 绿圆角 | ≥1 | 0 | 标签 |

## 5. 执行引擎逻辑

### 5.1 启动实例

1. 加载 `workflow_def.graph_json`
2. 找到 type=start 的节点
3. 沿出边到达第一个节点
4. 如果是审批节点 → 创建 `workflow_task` 待办
5. 设置 `instance.current_node_ids` = [当前节点ID]

### 5.2 审批通过 (advance)

1. 完成当前 task
2. 从当前节点沿出边找下一节点
3. 根据下一节点类型分流:
   - **审批节点** → 创建新 task, 更新 current_node_ids
   - **判断节点** → 评估条件表达式, 沿匹配分支前进
   - **并行节点** → 所有出口同时激活 (多个 current_node_ids)
   - **汇聚节点** → 检查所有入口是否到齐, 全部到齐才推进
   - **结束节点** → instance.status = APPROVED
   - **子流程节点** → 递归启动子流程实例, 完成后继续

### 5.3 审批驳回

instance.status = REJECTED, 所有 pending task 标记为 REJECTED

### 5.4 条件评估

从 `instance.context_json` 取值, 对判断节点的 `conditionExpr` 执行 SpEL 评估:
- `amount > 100000` → context.amount > 100000
- `bizType == 'PURCHASE'` → context.bizType == 'PURCHASE'

### 5.5 并行处理

- 并行节点激活: `current_node_ids = "branchA_approve_id,branchB_approve_id"`
- 汇聚节点 mergeMode=ALL: 等待所有入口 branch 的 task 都完成
- 汇聚节点 mergeMode=ANY: 任一入口到达即推进

## 6. 前端设计

### 6.1 流程设计器 WorkflowDesigner.vue

路由: `/approval/designer/:defId?`

三栏布局:
- **左侧 (120px)**: 7 种可拖拽节点
- **中间 (flex)**: Vue Flow SVG 画布，支持缩放、拖拽、连线
- **右侧 (220px)**: 节点属性编辑面板，根据选中节点类型动态切换

操作:
- 从左侧拖拽节点到画布
- 从节点 handle 拖出连线
- 点击节点 → 右侧显示属性面板
- 点击连线 → 可编辑条件标签(判断分支的 是/否)
- Delete 键删除选中节点/连线
- 保存 → 序列化 Vue Flow 实例为 graph_json → POST/PUT API

### 6.2 其他页面适配

- **DefList.vue**: 列表页, 新建 → 跳转设计器, 编辑 → 跳转设计器
- **PendingList.vue**: 适配 `/api/workflows/tasks/pending`
- **MyRequestList.vue**: 适配 `/api/workflows/tasks/my-requests`
- **HistoryList.vue**: 保持不变, 仍可查看旧审批历史
