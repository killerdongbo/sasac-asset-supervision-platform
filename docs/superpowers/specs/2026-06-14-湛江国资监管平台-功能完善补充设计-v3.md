### 3.0 资产生命周期管理

资产详情页应展示该资产从入库到当前的完整历史轨迹，实现"一资产一档案"。

#### 生命周期事件类型

| 事件类型 | 事件编码 | 数据来源 |
|---------|---------|---------|
| 入库登记 | STOCK_IN | stock_in 表 |
| 标签生成 | LABEL_GEN | asset_label 表 |
| 领用分配 | ASSIGNMENT | asset_assignment 表 |
| 调拨转移 | TRANSFER | asset_transfer 表 |
| 巡检记录 | INSPECTION | inspection_record 表 |
| 维修保养 | MAINTENANCE | maintenance_record 表 |
| 盘点确认 | INVENTORY | inventory_detail 表 |
| 折旧计提 | DEPRECIATION | depreciation_record 表 |
| 价值变动 | VALUE_CHANGE | asset_change_record 表 |
| 处置报废 | DISPOSAL | asset_disposal 表 |
| 审批流程 | APPROVAL | approval_record 表 |

#### 数据模型

```sql
CREATE TABLE asset_lifecycle_event (
    id BIGSERIAL PRIMARY KEY,
    tenant_id BIGINT NOT NULL,
    asset_id BIGINT NOT NULL,
    event_type VARCHAR(30) NOT NULL,
    event_title VARCHAR(200) NOT NULL,     -- 如"调拨至子公司A"
    event_detail JSONB,                    -- 事件详情（不同类型结构不同）
    source_table VARCHAR(50),              -- 来源表
    source_id BIGINT,                      -- 来源记录ID
    operator_id BIGINT,
    operator_name VARCHAR(50),
    event_time TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_lifecycle_asset ON asset_lifecycle_event(tenant_id, asset_id, event_time DESC);
```

#### 事件写入策略

采用**事件溯源**方式，在各业务操作完成后自动写入生命周期事件：

```
方案：AOP切面 + 事件监听器
  - 各 Service 完成业务操作后发布 Spring Event
  - AssetLifecycleListener 监听事件并写入 asset_lifecycle_event 表
  - 好处：业务代码无侵入，统一管理事件格式
```

#### API 设计

```
GET /api/assets/{id}/lifecycle          -- 获取资产生命周期时间线
    入参：page, size, eventType(可选筛选)
    返回：{content: [{eventType, eventTitle, eventDetail, operatorName, eventTime}], total}

GET /api/assets/{id}/lifecycle/summary  -- 生命周期摘要
    返回：{totalEvents, firstEvent, lastEvent, daysInService, maintenanceCount, transferCount}
```

### 3.1 数据导出模块

#### 交互流程

```
用户在列表页点击"导出" → 弹出参数确认对话框（筛选条件/格式选择）
  → 前端 POST /api/exports → 后端创建异步任务入队
  → 列表页 toast 提示"导出任务已创建"
  → 用户进入"导出中心"查看进度
  → 任务完成后点击下载（GET /api/exports/{id}/download → MinIO 预签名URL重定向）
```

#### 支持的导出场景

| 导出类型 | 触发位置 | 格式 | 数据源 |
|---------|---------|------|--------|
| 资产台账清单 | AssetList 页 | Excel | Asset 表（含分类/位置/责任人） |
| 资产流转记录 | 各流转列表页 | Excel | StockIn/Assignment/Transfer/Disposal |
| 盘点差异报告 | InventoryDiff 页 | Excel+PDF | InventoryDiff + 汇总统计 |
| 巡检任务报告 | InspectionTask 详情 | PDF | InspectionRecord + Anomaly |
| 折旧明细表 | DepreciationList 页 | Excel | DepreciationRecord |
| 19类监管报表 | ReportDetail 页 | PDF | ReportSnapshot |

#### 关键设计决策

- 小数据量（<1000行）同步导出，直接返回文件流
- 大数据量（≥1000行）走异步任务，轮询状态
- 文件保留7天自动清理（定时任务扫描 MinIO）
- 并发控制：每个租户同时最多5个导出任务

#### 错误处理

- 导出超时（10分钟）→ 标记 FAILED，记录错误信息
- MinIO 不可用 → 重试3次，仍失败则标记 FAILED
- 数据量过大（>10万行） → 拒绝并提示缩小筛选范围

#### 数据模型

```sql
CREATE TABLE export_task (
    id BIGSERIAL PRIMARY KEY,
    tenant_id BIGINT NOT NULL,
    export_type VARCHAR(50) NOT NULL,    -- ASSET_LIST/REPORT_PDF/INVENTORY_REPORT
    file_name VARCHAR(200),
    file_path VARCHAR(500),              -- MinIO 路径
    status VARCHAR(20) DEFAULT 'PENDING', -- PENDING/PROCESSING/COMPLETED/FAILED
    params JSONB,                         -- 导出参数（筛选条件）
    total_rows INT,
    error_message TEXT,
    created_by BIGINT,
    created_at TIMESTAMP DEFAULT NOW(),
    completed_at TIMESTAMP
);
```

#### API 设计

```
POST /api/exports                    -- 创建导出任务
GET  /api/exports                    -- 查询导出历史
GET  /api/exports/{id}/download      -- 下载导出文件
```

#### 前端页面

- `web/src/views/export/ExportCenter.vue` — 导出中心/下载管理
- 各列表页增加"导出"按钮（AssetList、InventoryDiffList 等）

#### 4.1.0 资产统计 — 集团穿透式汇总

支持按集团/公司维度进行资产统计，输入任意组织ID，自动汇总该组织及其所有下属公司的资产数据。

**API 设计：**

| API | 说明 |
|-----|------|
| `GET /api/statistics/assets` | 按组织层级汇总资产统计 |
| 入参 | orgId（必填，集团或公司ID）、assetCategoryId（可选）、startDate/endDate（可选） |
| 返回 | totalCount, totalOriginalValue, totalNetValue, childOrgStats:[] |

**组织穿透逻辑：**

```
输入 orgId
  → 递归查询 organization 表（parent_id 树结构）获取所有下属组织ID列表
  → SELECT ... FROM asset WHERE org_id IN (本级 + 所有下属) GROUP BY org_id
  → 按层级聚合：本级汇总 + 各下属公司明细
```

**返回结构示例：**

```json
{
  "orgId": 1,
  "orgName": "湛江市国资委",
  "totalCount": 12580,
  "totalOriginalValue": 850000000.00,
  "totalNetValue": 620000000.00,
  "categoryDistribution": [
    {"category": "房屋建筑物", "count": 320, "value": 450000000.00},
    {"category": "通用设备", "count": 5800, "value": 120000000.00}
  ],
  "childOrgStats": [
    {
      "orgId": 10,
      "orgName": "某集团公司",
      "totalCount": 3200,
      "totalOriginalValue": 280000000.00,
      "children": [
        {"orgId": 101, "orgName": "子公司A", "totalCount": 1200, ...},
        {"orgId": 102, "orgName": "子公司B", "totalCount": 2000, ...}
      ]
    }
  ]
}
```

**前端交互：**

- 顶部组织树选择器（支持搜索），选择后刷新整页统计
- 汇总卡片（总数/总原值/总净值）
- 下属组织对比柱状图（按金额排序）
- 支持点击下钻：集团 → 子公司 → 具体资产列表

**性能优化：**

- 组织树缓存 Redis（变更时失效）
- 大集团（>50家下属）启用异步预计算（每日凌晨汇总写入 stats_cache 表）

#### 动态表单渲染架构

```
报表模板（report_template）
  ├── template_config: JSONB    ← 表格列定义、分组、合计行规则
  ├── field_config: JSONB       ← 每个字段的类型/校验规则/字典引用
  └── validation_rules: JSONB   ← 勾稽关系定义

前端 DynamicReportForm.vue
  ├── 读取 template_config → 生成 <el-table> 列
  ├── 读取 field_config → 生成 <el-form-item> 输入控件
  └── 读取 validation_rules → 提交前执行三层校验
```

#### 三层校验引擎

```
第一层：字段级校验
  - 必填/数据类型/范围/正则
  - 如：面积 > 0，金额精度 ≤ 2位小数

第二层：表内校验（行间关系）
  - 合计行 = SUM(明细行)
  - 净值 = 原值 - 累计折旧
  - 如：累计折旧 ≤ 原值

第三层：表间校验（跨报表）
  - 汇总表.土地总值 = Σ(土地明细表.金额)
  - 对外投资合计 = 股权投资 + 债权投资
  - 总资产 = 各类资产合计
```

### 6.1 合并报表智能辅助

#### 三维匹配引擎

内部交易识别采用三维匹配：

```
维度1：金额匹配（容差≤0.01元）
  A公司应收B → B公司应付A → |差额| ≤ 0.01

维度2：对手方匹配
  A的交易对手=B → B的交易对手=A → 双向确认

维度3：日期匹配（容差≤3天）
  A记录日期 与 B记录日期 差异 ≤ 3天

三维全部命中 → 自动匹配（置信度 HIGH）
两维命中 → 建议匹配（置信度 MEDIUM，人工确认）
一维命中 → 待核实（置信度 LOW）
```

#### 抵销分录自动生成

匹配成功后自动生成抵销分录建议：
- 内部应收/应付抵销
- 内部收入/成本抵销
- 内部投资/权益抵销
- 内部利润/未分配利润抵销

人工审核确认后执行。

## 七、技术实现约束

### 7.1 通用约束

- 所有表必须包含 `tenant_id` 字段（多租户隔离）
- 所有写操作通过 TenantContext 自动注入 tenant_id
- 分页接口统一返回 `{content, total, page, size}` 格式
- 金额字段统一 `DECIMAL(18,2)`，不使用 FLOAT
- 时间字段统一 `TIMESTAMP`，存储 UTC，前端按时区展示
- 删除操作统一软删除（`deleted` 字段）

### 7.2 性能约束

- 列表页接口响应 ≤ 500ms（P95）
- 导出任务 ≤ 10分钟超时
- Redis 缓存 TTL：统计类 5min，配置类 30min
- 大表（>100万行）必须有联合索引覆盖常用查询

### 7.3 安全约束

- 所有 API 必须经过 JWT 认证
- 数据操作必须校验 tenant_id 归属
- 敏感操作（删除/导出/授权）记录审计日志
- 密码/密钥禁止硬编码，统一使用环境变量
