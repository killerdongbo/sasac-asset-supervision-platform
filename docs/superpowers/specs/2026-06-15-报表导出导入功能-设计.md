# 报表导出/导入功能 — 设计文档

> 版本：v1.0  
> 日期：2026-06-15  
> 状态：已确认，待实施

---

## 1. 概述

在统计报表模块中新增报表导出/导入功能，支持 19 种国资监管标准报表的 Excel 导出、导入以及模板下载。

### 1.1 核心功能

- **报表导出**：选择报表类型 → 组织/周期 → 异步生成 Excel → 下载
- **报表导入**：下载模板 → 线下填报 → 上传校验 → 预览 → 入库
- **模板下载**：19 种报表标准 Excel 模板（含表头 + 示例行 + 填写说明）

### 1.2 设计原则

- 复用现有 `export_task` 表、`ExportService` 异步框架、EasyExcel 技术栈
- 19 个报表统一走 `exportType` 枚举分发，每个报表一个私有方法
- Excel 文件上传 MinIO，下载返回预签名 URL
- 导入采用逐行校验 + 事务写入，校验失败全部不入库

## 4. 前端设计

### 4.1 路由与入口

- 路由：`/reports/export`
- 菜单入口：统计报表 → 报表导出

### 4.2 页面结构（三 Tab）

## 5. 后端服务设计

### 5.1 ExportService（修改）

每个报表一个 private 方法，统一签名模式：

```java
private void exportXxx(ExportTask task) {
    // 1. 解析 task.params → orgId, period, filters
    // 2. 查询数据：assetMapper / inspectionMapper / financialMapper ...
    // 3. EasyExcel.write() → 临时文件 .xlsx
    // 4. MinIO upload → 得到 objectKey
    // 5. 更新 task: filePath=objectKey, totalRows, status=COMPLETED
}
```

`executeExportAsync` 中扩展 switch 分发：

```java
switch (task.getExportType()) {
    case BALANCE_SHEET        → exportBalanceSheet(task);
    case ASSET_BASE_LIST      → exportAssetBaseList(task);
    case PROBLEM_ASSET_LIST   → exportProblemAssetList(task);
    case REVITALIZATION_LIST  → exportRevitalizationList(task);
    // ... 共 19 个 + 填充 3 个已有桩方法
}
```

### 5.2 ReportImportService（新增）

导入流水线：

```
Step 1: validateTemplate(file, reportType)
        → 校验上传文件表头列是否匹配模板定义

Step 2: parseAndValidate(file, reportType)
        → EasyExcel 逐行读取，校验格式/必填/数值范围
        → 返回 RowErrors: [{row, field, reason}]

Step 3: preview(parsedData, page, size)
        → 返回分页预览（前 20 行）

Step 4: importData(importBatch)
        → @Transactional 写入对应业务表
```

## 6. 错误处理

### 6.1 导出错误

| 场景 | 处理 |
|------|------|
| 数据源为空 | 生成仅有表头的 Excel + 提示文字 |
| 查询超时 | task.status=FAILED, errorMessage='查询超时，请缩小筛选范围后重试' |
| MinIO 上传失败 | 重试 3 次，均失败则 task.status=FAILED |
| 并发限制 | 单租户最多 5 个 PENDING/PROCESSING 任务（已有逻辑） |

### 6.2 导入错误

| 场景 | 处理 |
|------|------|
| 模板不匹配 | HTTP 400 + "请使用正确的报表模板" |
| 数据校验失败 | 返回 `ImportError[]`: [{row, field, reason}] |
| 入库失败 | @Transactional 回滚，全部不入库 |
| 文件格式错误 | HTTP 400 + "仅支持 .xlsx 格式" |

|----------|---------|------|
| 单元测试 | 每个 export 方法（Mock Mapper），导入校验逻辑，模板匹配 | ≥ 80% 行覆盖率 |
| 集成测试 | ExportController → Service → PostgreSQL (Testcontainers) → MinIO | 每个报表核心流程 |
| Excel 验证 | Apache POI 读取生成文件，验证表头、行数、汇总行公式 | 每种报表至少 1 条 |
| 导入流程测试 | 正常导入 / 模板不匹配 / 校验失败 / 事务回滚 | 完整覆盖 |

## 8. 实施优先级

| 阶段 | 内容 | 预计 |
|------|------|------|
| Phase 1 | 后端 19 个导出方法 + 前端导出中心（导出 Tab） | 优先 |
| Phase 2 | 19 个模板文件 + 模板下载 Tab | 优先 |
| Phase 3 | 报表导入服务 + 前端导入 Tab | 次优先 |
