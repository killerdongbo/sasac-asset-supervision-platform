# 测试补全实施计划

> **目标：** 为 Phase 1+2 新增的 12 个 Service 类补写单元测试，覆盖核心业务逻辑。

**策略：** 每个 Service 写 3-5 个关键测试用例（创建成功/参数校验/查询过滤/异常场景），使用 Mockito + JUnit 5 + AssertJ。

**工期：** 1 个执行批次，分 2 个子智能体并行编写。

## 测试模板

每个测试类遵循 Phase 1 已建立的 TDD 模式（参考 `AssetServiceTest.java`）：

```java
@ExtendWith(MockitoExtension.class)
class XxxServiceTest {

    @Mock private XxxMapper mapper;
    @InjectMocks private XxxService service;

    // 正常场景: shouldCreateXxx
    // 异常场景: shouldRejectDuplicate / shouldThrowWhenNotFound
    // 查询场景: shouldQueryWithFilters
    // 边界场景: shouldHandleEdgeCase
}
```

无占位符，每个测试包含完整代码。
