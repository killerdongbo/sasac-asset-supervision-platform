# Phase 2 六升级模块 + 移动端重做 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在一期资产管理和 Phase 1 四新模块基础上，串行交付产权管理 → 财务监督 → 业绩考核与薪酬 → 监督追责 → 重大事项管理 → 预警驾驶舱升级 + 移动端重做，共 6 个里程碑。

**Architecture:** 继承 Phase 1 的 SpringBoot 3 + MyBatis-Plus + PostgreSQL 单 Schema 多租户架构。新模块代码按独立包结构（`property/`、`finance/`、`performance/`、`supervision/`、`majorevent/`）组织，复用一期 RBAC、审批引擎、审计日志基础设施。预警驾驶舱在一期已有 dashboard 上增强；移动端在一期 H5 基础上重构为独立前端项目。

**Tech Stack:** SpringBoot 3, MyBatis-Plus, PostgreSQL 15, Vue 3, TypeScript, Element Plus, ECharts 5, uni-app (移动端)

**工期：** 12 周，后端 1 人 + 前端 1 人，串行开发。
