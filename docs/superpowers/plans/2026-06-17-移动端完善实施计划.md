# 移动端完善实施计划

> **状态：** 直接执行 | **工期：** 1个实施批次

**目标：** 将 M11 移动端从静态原型升级为可构建运行的完整 uni-app 应用，补全 API 层、认证、状态管理。

**当前缺口（代码审查发现）：**
- 无 `utils/request.js`（HTTP 客户端）
- 无 `store/`（状态管理）
- 无 `api/`（API 模块）
- 无登录流程
- `package.json` 依赖缺失
- TabBar 配置错误
- 图标资源缺失
- 8 个页面数据全部硬编码

---

## Task 1: 基础设施搭建

### `mobile/utils/request.js`
uni.request 封装，功能对应 `web/src/api/client.ts`：
- baseURL 配置
- 请求拦截器：从 store 读取 token → Authorization Bearer
- 请求拦截器：从 store 读取 tenantId → X-Tenant-Id header
- 响应拦截器：自动解包 ApiResponse（success/error/data/meta）
- 401/403 → 清除 token → 跳转登录页
- 超时 30s

### `mobile/store/index.js`
Pinia store：
- `useAuthStore` — token, userInfo, tenantId, orgId 的 getter/setter
- `login(username, password)` action
- `logout()` action
- 持久化到 `uni.getStorageSync`

### `mobile/api/` 目录
从 `web/src/api/` 复制并调整为 uni-app 格式（将 `client.get/post/put/delete` 替换为 `request` 模块调用）：
- `api/auth.js` — login, logout, getUserInfo
- `api/hr.js` — 人员 CRUD
- `api/approval.js` — 待审批列表/审批操作
- `api/project.js` — 项目列表/进度填报
- `api/investment.js` — 投资组合查询
- `api/finance.js` — 财务快报
- `api/alert.js` — 预警消息
- `api/decision.js` — 三重一大审批

### `mobile/pages/login/login.vue`
登录页面：
- 用户名 + 密码输入框
- 登录按钮 → 调用 authStore.login → 成功后跳转首页
- 记住密码功能

## Task 3: 页面接入真实 API

逐个改造 8 个页面，将硬编码数据替换为 API 调用：

| 页面 | 改动 |
|------|------|
| `pages/index/index.vue` | 调用多模块 API 获取待办数量（角标） |
| `pages/approval/list.vue` | 调用 `/api/approval/pending` 获取真实待审批列表 |
| `pages/hr/salary.vue` | 调用 `/api/hr/salary` 获取工资条 |
| `pages/decision/approve.vue` | 调用 `/api/decision/items/{id}` + 提交审批结果 |
| `pages/project/progress.vue` | 调用 `/api/projects` + `/api/projects/{id}/progress` |
| `pages/invest/portfolio.vue` | 调用 `/api/investment/projects` |
| `pages/finance/snapshot.vue` | 调用 `/api/finance/indicators` |
| `pages/alerts/index.vue` | 调用 `/api/alerts` |
